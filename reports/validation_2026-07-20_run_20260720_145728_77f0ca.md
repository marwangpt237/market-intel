## How Validation Works

```
Every claim becomes a first-class object:
  { id, entity, claim_type, value, sources, evidence_count,
    first_seen, last_seen, last_verified, supporting_evidence,
    contradicting_evidence, confidence_score, validation_status,
    expiration_date, version_history }

Validation rules:
  1. Minimum independent sources (default 2 for PROBABLE, 3 for VERIFIED)
  2. Source reliability weighting (TrustLayer)
  3. Contradiction detection (diff > 30% tolerance → CONFLICTED)
  4. Confidence = Σ(reliability × evidence_confidence) / total
     + diversity_bonus - contradiction_penalty
  5. Staleness: last_seen > expiration_date → EXPIRED
  6. Status: UNKNOWN → HYPOTHESIS → PROBABLE → VERIFIED
                                          ↓
                                       CONFLICTED / EXPIRED

Every decision references the Claim IDs that justify it.
If a recommendation depends on weak/conflicting claims, confidence is reduced
and warnings are added to the Decision Ledger entry.
```
