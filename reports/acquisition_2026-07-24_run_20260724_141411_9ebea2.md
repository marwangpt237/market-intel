## How the Planner Works

```
1. Knowledge Gap Detector aggregates missing-evidence requests from Validation Engine
   - Groups by (entity, claim_type)
   - Computes priority: P0 (high impact + urgent) → P3 (minimal)

2. Data Acquisition Planner generates concrete actions per gap:
   - crawl: collect N items from source X with query Y
   - increase_frequency: collect every N hours instead of default
   - historical_collection: pull N days of historical data
   - search_corroboration: search additional sources for cross-validation
   - schedule_validation: re-run Validation Engine in N hours

3. Each plan estimates:
   - evidence_gain (how many new evidence pieces expected)
   - confidence_lift (how much target confidence will improve)
   - cost (resource units: API calls + time)
```
